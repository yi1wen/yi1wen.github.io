设计模式

发表于 2024-07-10

观察者模式

一对多，被观察者 & 观察者 , 重点是被观察者
被观察者需要维护 订阅者列表 ，并且需要通知他们

发布订阅

多对多，发布者（消息） & 订阅者，重点是发布订阅中心
发布订阅中心不是收集发布者和订阅者，而是收集 发布者的消息 和 订阅者的通知方式 ，分别对应两个对象，用同一个 key 串起来找到
发布者的发布意思是 增加消息，订阅中心的发布才是 根据订阅者的通知方式，用消息通知他们

  1. 发布者：增加消息
  2. 订阅者：订阅 + 规定同通知方式
  3. 发布订阅中心：收集发布者的消息 和 订阅者的通知方式，用消息 + 通知方式 通知订阅者

Map、Set、Object

发表于 2024-07-10

Map

  1. 有序
  2. key 任意类型 - 不知道key具体类型时用
  3. [key value] 形式
  4. api: set, delete, has, size, clear,get
  5. 遍历：[key, value] 解构，forEach解构转数组
  6. 内部用了哈希表

Set 集合

  1. 有序
  2. [value, value] 形式
  3. value 任意类型，唯一值
  4. 精确相等运算符：=== + NAN==NAN, undefined -- undefined
  5. api: add, delete, has, delete, clear

Object

  1. 无序
  2. key 为字符串/ Symbol - 发生类型转换，获得Symbol 键用get
  3. 删除 有性能问题
  4. 支持 JSON 序列化
  5. 遍历：Object.keys（长度）, Object.Values, Object.entries, 不能用遍历操作

Map 和 Object 的 区别

  1. 增加键：量大时Map快，key未知类型时，Map更适用
  2. 删除键：Map 好
  3. 遍历时：Map 有序，自带，Object 无序
  4. 大量键时： Map快
  5. Json：Object才行

如何确定一个类型支不支持迭代

typeof obj[Symbol.iterator]

判断某个元素在不在 Object 里

  1. obj.key == undefined
  2. key in obj

Webpack和Vite

发表于 2024-07-10

Webpack

  1. 从入口开始
  2. 依赖分析
  3. 模块转换
  4. 打包输出

从入口开始分析项目的依赖关系
构建整个项目的依赖关系图，依赖关系树
根据各种loader将各个模块转换为浏览器可理解的模式
将所有模块打包成输出文件

Vite

  1. 静态服务器
  2. 按需编译
  3. 热模块替换
  4. 生产建构

利用静态服务器，利用原生ES模块，将每一个模块作为独立的Http请求提供
请求哪个模块提供哪个模块，用ESbuild即时编译
更新页面无序刷新，Vite将更新的模块给浏览器，浏览器用HMR技术局部更新
生产环境用Rollup打包，更快的加载速度和更小的文件体积

区别

  1. 构建方式：Webpack用plugin 和 loader实现各种功能，输出打包文件；Vite 用 原生ES
  2. 时间：所以Webpack 较长时间
  3. 性能：Webpack 适用于大型项目的优化，Vite 实时预览的小型项目

线程与进程

发表于 2024-07-10
  1. 进程包含线程，有独立的内存空间和文件描述符
  2. 同一个进程的多个线程共享一份地址空间和文字描述符号，一个线程挂了，会影响到其他线程，可能会带走整个进程
  3. 进程是操作系统资源分配的基本单位，线程是操作系统调度的基本单位

线程之间的通信

  1. onmessage()
  2. postMessage()
  3. 全局变量

进程之间的通信

  1. send()
  2. 管道：用于，分为命名管道和匿名管道（父子进程）。内核在内存创建一个缓存区
  3. 消息队列：存放在内存中的消息链表，消息就是用户自定义的数据结构
  4. 共享内存：页表是将进程的逻辑地址映射到物理空间的表，共享内存就是将两个进程的逻辑地址通过页表映射到同一个物理空间
  5. 信号：异步通信机制。发送信号来通知进程某个异步事件的发送，进程终端处理程序后，进程才恢复执行
  6. Socket 套接字：与不同主机上的进程通信

多进程与并发

  • 负载均衡 -> 轮叫调度：CPU 和 I/O 影响繁忙度，由主进程接收连接，分发给工作进程
  • 状态共享：第三方数据存储，改变数据，通知更新，状态同步的方式是轮询或主动通知（由一个通知进程进行轮询获得数据并更新）

OSI七层模型

发表于 2024-06-23
  1. 应用层：HTTP，DNS，FTP（文本），SMTP（邮件）
  2. 表示层：翻译成机器语言，安全加密，压缩等
  3. 会话层：SSL，TLS
  4. 传输层：TCP，UDP
  5. 网络层（包）：IP，实现不同网络的路径选择
  6. 数据链路层（帧）：MAC
  7. 物理层（比特流）

数据封装过程

  1. 应用层：数据
  2. 传输层：数据 + TCP头部
  3. 网络层：数据 + TCP头部 + IP头部
  4. 数据链路层：数据 + TCP头部 + IP头部 + MAC头部

发表于 2024-06-23

var,let,const的区别

  1. var 变量提升，let const无，所以会出现暂时死区
  2. var 函数作用域，let const 块级作用域
  3. var 可以重复声明，let const 不能
  4. const 必须初始化，不能重复赋值，指的是引用地址不能改变

箭头函数

  1. 没有自己的 this，往上找 this， 使用call,apply,bind并不会改变箭头函数中的this指向。
  2. 不能做构造函数，不能被 new，没有原型 prototype 属性,
  3. 没有 arguments, super
