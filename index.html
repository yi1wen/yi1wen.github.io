flex

MDN：flex的解释

  1. flex-grow 当容器还有剩余空间放大的比例
  2. flex-shrink 当容器没有剩余空间缩小的比例
  3. flex-basis 默认大小，以上两个是通过此值来计算是否放大或缩小的
  4. flex: [flex-grow] [flex-shrink] [flex-basis] 原始值：flex:0 1 auto
  5. flex-direction 主轴方向 row-行 col-列
  6. flex-wrap 指定单行显示还是多行显示
  7. flex-flow: [flex-direction] [flex-wrap]

position

  1. static 正常
  2. relative 原始位置用空白占位，对于原始位置的偏移
  3. absolute 移出正常文档流，对最近的非 static 的祖先元素的右上角点的定位
  4. fixed 移出正常文档流，对屏幕视口的定位，滚动时不会改变，当元素祖先的 transform、perspective、filter 或 backdrop-filter 属性非 none 时，容器由视口改为该祖先。
  5. sticky

跨域

发表于 2024-02-25

为什么

浏览器同源策略：同协议，域名，端口的资源
限制：

  1. 无法操作彼此的 DOM 元素
  2. 无法共享缓存，如 cookie 和 localstorage

JSONP

  1. 定义回调函数名称 fn
  2. 通过 script 标签的 src 属性请求接口数据 （script 标签不熟浏览器同源策略影响），指定回调函数为 fn，客户端将数据封装到 fn 函数调用后返回。

CORS

设置 Access-Control-Allow-Origin 字段允许前端访问

nginx 反向代理

正向代理：访问方主动设置，访问方知道，隐藏了真实的客户端
反向代理：被访问方设置的，访问方无感知，隐藏了真实的服务器

反向代理有负载均衡的好处，同时将对真实服务器的请求反向代理到本机服务器，从而避免浏览器“同源策略限制”

算法：排序

发表于 2024-02-23 | 分类于 算法

排序

稳定性

稳定性是如果有多个相同元素，经过排序，它们的位置是被控制的，就叫稳定；若它们的位置是随机的，就是不稳定

算法

| 排序算法 | 稳定度 | 时间复杂度 | 额外空间 | 适用情况 |
| 冒泡排序 | 稳定 | O(n2) | O(1) | n小 |
| 快速排序 | 不稳定 | O(nlogn) ~ O(n2) | O(nlogn) | n大 |
| 归并排序 | 稳定 | O(nlog) | O(1) | n大 |

冒泡排序

一循环一个，循环中两两对比，交换位置，大的下沉，小的“冒泡”

快速排序

挖坑 + 分治
基准值，小值区和大值区的分界线，分区，递归再排序

归并排序

两两排序合并成一个，合并后成为一个，再和其他两两排序合并

Vue3源码：响应式原理之effect和Dep的结构

发表于 2024-02-22 | 分类于 源码 ， Vue3

总结

  1. 是什么：用 ReactiveEffect 将 fn 函数包装成 effect，用 Proxy 代理 原始对象 变成 响应式数据
  2. 收集：fn 访问 响应式数据，在 Proxy 中 响应式数据的 依赖集合Dep 收集 effect，同时 effect 收集 响应式的依赖集合Dep
  3. 触发：依赖 的意思是 fn 依赖 响应式数据，响应式数据 影响 fn。当响应式数据改变时 说明会影响到 fn， 所以要触发 fn，让 fn 更新状态；
  4. 依赖更新：fn 更新状态时，需要的数据可能会改变，也会重新访问需要的 响应式数据，effect 收集 Dep 目的是订阅，Dep 收集 effect 的 目的是取消订阅

Vue3源码：ref

发表于 2024-02-21 | 分类于 源码 ， Vue3

reactive 用于对象的响应式方案，
ref 用于 原始值的响应式方案，
原因有2：原始值的赋值为原始值，对象的赋值为内存地址，也就导致原始值的形参和实参之间无引用关系，完全独立；Proxy无法对原始值代理

ref 的实质是，用对象上的 value 属性代理原始值，再将该对象用 reactive Proxy代理
