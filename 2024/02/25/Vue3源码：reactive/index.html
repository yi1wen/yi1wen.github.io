总结

reactive 就是根据不同的对象类型(rawType -> TargetType)，使用不同的处理器 handler，不同处理器里有一一对应的操作方法，将对象进行 Proxy 代理，并记录到 reactiveMap 中，返回代理对象;
Proxy 对应拦截操作的核心 api 是 Refelct 对应的方法，使用 Reflect 的原因是 receiver 参数使触发对象劫持时保证正确的 this 指向
