Http请求头和响应头

发表于 2024-02-27

文件链接再浏览器中打开是预览还是下载

  • 预览：Content-type: application/pdf;Content-Disposition: inline
    -下载：Content-type: application/octet-stream 二进制流直接下载、Content-type: application/pdf ; Content-Disposition: attachment

Content-Type: type(类型)/subtype(子类型)

  • type: application(二进制数据),model(三维模型数据),audio,font,image,video,text; 多部分类型 message,multipart
  • application/octet-stream 未知的二进制文件
  • text/plain 未知的文本文件：text/css,text/html,text/javascript
  • multipart/form-data 用于 HTML 表单从浏览器发送信息给服务器。
  • multipart/byteranges 用于指出这个文件由若干部分组成，每一个都有其请求范围 206 + Content-Range

缓存

  • 强缓存：Empires，Cache-Control
  • 协商缓存：Etag，Last-Modified

跨域

  • Access-Control-Allow-Origin

Cookie

  • Cookie,set-Cookie

如何保证 Cookie 被安全发送

  • Secure 属性：通过 https 发送
  • HttpOnly：无法使用 js 读取到 cookie，就无法恶意执行， 用于防范 XSS
  • Domain：指定接受 Cookie 的主机
  • Path：指定请求的路径
  • SameSite：是否随跨站请求一起发送，防止 CSRF

Session 和 Cookie 的区别

  • Session 存储在服务端，Cookie 存储在客户端，都是存储用户信息的地方
  • Http 是无状态协议， Session 用来弥补这个缺陷，每个用户有各自的 Session，存储状态信息，唯一标识为 sessionid，所以 Session 是用户状态与信息的内容
  • 使用 Cookie 机制来在每次 Http 请求的时候带上 sid，还可以自动登录，是一个存储用户状态与信息的容器

请求方法

简单请求

  1. GET,HEAD,POST(Content-Type: application/x-www-form-urlencoded, multipart/form-data, text/plain, 也就是表单数据，未知文本文件)
  2. 无自定义 HTTP 头部

OPTIONS

  • OPTIONS 请求允许通信选项：CORS 中的预检请求，以检测实际请求是否可以被服务器所接受
  • 请求原因：复杂请求，跨域，自定义请求头
  • 如何避免：1。使用代理避免跨域 2.简单请求 3.不自定义请求头

原型和原型链

发表于 2024-02-26

每一个实例函数的 __proto__ 属性指向构造函数的 prototype 属性；
每一个构造函数（包括 Object 函数）都是是 Function 函数的实例函数，所以每个构造函数上有 __proto__ 属性，指向 Function 函数的 prototype 属性；
每一个 prototype 属性默认是一个空对象，是 Object 函数的实例函数，所以 prototype 属性上有 __proto__ 属性，指向 Object 函数的 prototype 属性；
Object 的 prototype 属性的 __proto__ 属性指向 null 是原型链的终点。

浅拷贝与深拷贝

发表于 2024-02-26

赋值

对于引用类型数据来说，赋值是复制引用类型数据的堆内存地址

浅拷贝

对于引用类型数据来说，浅拷贝是创建一个新的对象，对第一层属性进行拷贝，若有深层的引用类型数据，则还是指向同一内存地址，如 拓展运算符，slice，concat，assign

深拷贝

对于引用类型数据来说，深拷贝就是让两个引用类型数据完全独立，递归进行浅拷贝，深层的引用类型数据也指向不同内存地址，但是属性进行精确拷贝，如 _.cloneDeep, jQuery.extend, JSON.stringify（会忽略 undefined, symbol, function）

前端攻击

发表于 2024-02-25

CSRF 跨站请求伪造

跨站：在被攻击网站上获得了 cookie，诱导进入第三方

请求伪造：利用用户已经在被攻击网站上获得的 cookie，冒充用户执行操作

类型：get，post，链接

XSS 跨站脚本攻击

  • 存储型：恶意代码存储在服务器中，用户收到响应中包含恶意代码并执行
  • 反射型：恶意代码拼接在 url 上，服务器提取拼接在 HTML 上返回，
  • Dom 型：url 前端执行请求的路径

预防：防止攻击者提交恶意代码

CSS

发表于 2024-02-25 | 分类于 CSS

flex

MDN：flex的解释

  1. flex-grow 当容器还有剩余空间放大的比例
  2. flex-shrink 当容器没有剩余空间缩小的比例
  3. flex-basis 默认大小，以上两个是通过此值来计算是否放大或缩小的
  4. flex: [flex-grow] [flex-shrink] [flex-basis] 原始值：flex:0 1 auto
  5. flex-direction 主轴方向 row-行 col-列
  6. flex-wrap 指定单行显示还是多行显示
  7. flex-flow: [flex-direction] [flex-wrap]

position

  1. static 正常
  2. relative 原始位置用空白占位，对于原始位置的偏移
  3. absolute 移出正常文档流，对最近的非 static 的祖先元素的右上角点的定位
  4. fixed 移出正常文档流，对屏幕视口的定位，滚动时不会改变，当元素祖先的 transform、perspective、filter 或 backdrop-filter 属性非 none 时，容器由视口改为该祖先。
  5. sticky

形状

  1. 椭圆：长方形 + border-radius: 50% ; border-radius: 水平 / 垂直；border-radius: 左上 右上 右下 左下
  2. 平行四边形：transform: skewX(-45deg)
  3. 菱形：transfrom: rotate(45deg); clip-path: polygon(50% 0, 100% 50%,50% 100%, 0 50%)
  4. 切角：渐变{background: linear-gradient(-45deg, transparent 15px,)}
  5. 梯形：3D模拟{transform: perspective(.5em) rorateX(5deg)}

行内元素(inline)，块级元素(block)，行内块元素(inline-block)

区别：一是排列方式，二是宽高边距设置，三是默认宽度。

  • 行内元素：1.高度无效，padding，margin上下无效，行高 line-height 有效; 2.宽度无效，随内容而变化
  • 行内块元素: 一行中放置多个，宽度随内容，高度可设置

跨域

发表于 2024-02-25

为什么

浏览器同源策略：同协议，域名，端口的资源
限制：

  1. 无法操作彼此的 DOM 元素
  2. 无法共享缓存，如 cookie 和 localstorage

JSONP

  1. 定义回调函数名称 fn
  2. 通过 script 标签的 src 属性请求接口数据 （script 标签不熟浏览器同源策略影响），指定回调函数为 fn，客户端将数据封装到 fn 函数调用后返回。

CORS

设置 Access-Control-Allow-Origin 字段允许前端访问

nginx 反向代理

正向代理：访问方主动设置，访问方知道，隐藏了真实的客户端
反向代理：被访问方设置的，访问方无感知，隐藏了真实的服务器

反向代理有负载均衡的好处，同时将对真实服务器的请求反向代理到本机服务器，从而避免浏览器“同源策略限制”
