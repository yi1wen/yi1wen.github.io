继承

发表于 2024-06-22

总结

  1. 原型链继承
  2. 构造函数继承
  3. 组合继承
  4. 原型式继承
  5. 寄生式继承
  6. 寄生组合式
  7. extends

原型链继承

子类的构造函数的 prototype 原空对象，现赋值为父类构造函数的实例对象
child.__proto__ = Children.prototype, Children.prototype.__proto__ = Parent.prototype
缺点：只能继承方法，不能继承实例属性

构造函数继承

子类构造函数中用 call ，apply，调用父类构造函数，绑定实例属性的this为子类构造函数
缺点：只能继承实例属性，不能继承原型链

组合继承：原型链 + 构造函数

原型链缺点：只能继承函数，不能继承实例属性
构造函数缺点：只能继承实例属性，不能继承原型链

组合继承缺点：调用两次父类构造函数，造成不必要的开销

原型式继承

无构造函数，用Object.create，新建空对象作为子类对象，其原型指向父类对象
child.__proto__ = parent
缺点：子类实例无法向父类传参

寄生式继承：原型式 + 工厂函数

不需要单独创建构造函数
缺点：工厂函数中自定义添加属性，无法复用函数

寄生组合式

寄生缺点：无法复用为子类对象添加的函数
组合式缺点：调用两次父类构造函数
组合寄生式优点：只引用一次父类构造函数，继承父类原型链，实例属性，添加子类公共私有方法

http和https的区别

发表于 2024-06-22

总结

  1. https 是 http + ssl 层
  2. 数据完整性：摘要算法
  3. 数据保密性：非对称加密，公钥加密，私钥解密，不需要传输密钥，安全性增加（对称加密需要传递密钥）
  4. 身份校验安全性：数字签名（私钥加密，公钥解密，确保消息来源可信）和数字证书（确保公钥可信）

https 握手

  1. 浏览器：协议版本加密算法，压缩算法，随机数1
  2. 服务端：确定协议版本，加密算法，数字证书，公钥，随机数2
  3. 浏览器：验证证书，预主密钥，3个随机数组装会话密钥，用公钥加密预主密钥传过去
  4. 服务端：用私钥解密预主密钥，3个随机数组装会话密钥，会话密钥协商成功，双方用同一份会话密钥通信

http版本

发表于 2024-06-21

http 0.9

无响应头，请求头，只有GET

http 1.0

有响应头，有请求头，多种请求方法

http 1.1

持久连接

一个TCP连接上可以发送多个请求和响应

管线化

一个TCP连接上可以连续发送多个请求和响应
发送请求不必等待上一个请求的响应，但是需要按序发送响应，队头阻塞

分块传输编码

可以在不知道响应数据的完整长度的时候，分块逐步发送，而不必等待响应完全加载。
适用于在意外网络中断前，尽可能多地响应数据

http 2.0

多路复用

在一个TCP连接上，复用多个数据流

乱序发送响应，解决管线化的对头阻塞

二进制分帧

提高传输效率，加密算法升级，更安全
分成一个小小的二进制帧，帧头信息标志，头部压缩

服务器推送

服务器可以把客户端可能需要的数据一起响应回去，但也可能造成不必要的响应

http 3.0

基于UDP 的 QUIC协议

发表于 2024-06-18

总结

  1. hash 模式
  2. history 模式

hash 模式

‘#’号后面的字符串，不会重新加载页面，会触发 hashchange事件，获取hash值来修改页面路由
hash值改变，不会重新加载页面，而是在栈结构中添加和移除数据，实现回退和前进，并切换

history 模式

hash值改变，对服务器发送请求，原理：pushstate，replacestate
