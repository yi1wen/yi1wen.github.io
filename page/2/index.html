Vue3源码：响应式原理之effect和Dep的结构

发表于 2024-02-22 | 分类于 源码 ， Vue3

总结

  1. 是什么：用 ReactiveEffect 将 fn 函数包装成 effect，用 Proxy 代理 原始对象 变成 响应式数据
  2. 收集：fn 访问 响应式数据，在 Proxy 中 响应式数据的 依赖集合Dep 收集 effect，同时 effect 收集 响应式的依赖集合Dep
  3. 触发：依赖 的意思是 fn 依赖 响应式数据，响应式数据 影响 fn。当响应式数据改变时 说明会影响到 fn， 所以要触发 fn，让 fn 更新状态；
  4. 依赖更新：fn 更新状态时，需要的数据可能会改变，也会重新访问需要的 响应式数据，effect 收集 Dep 目的是订阅，Dep 收集 effect 的 目的是取消订阅

Vue3源码：ref

发表于 2024-02-21 | 分类于 源码 ， Vue3

reactive 用于对象的响应式方案，
ref 用于 原始值的响应式方案，
原因有2：原始值的赋值为原始值，对象的赋值为内存地址，也就导致原始值的形参和实参之间无引用关系，完全独立；Proxy无法对原始值代理

ref 的实质是，用对象上的 value 属性代理原始值，再将该对象用 reactive Proxy代理

发表于 2024-02-21 | 分类于 源码 ， Vue3

总结

KeepAlive 就是创建一个 KeepAlive 组件，插槽为需要 keepalive 的组件。

  1. 在 KeepAlive 组件中主要做：创建一个不渲染的 div 隐藏容器，用来将组件所渲染的内容移动进去隐藏；创建失活和激活的函数；缓存 cache (Map 结构的对象)：key：vnode.type, value: vnode
  2. 在需要 keepalive 的组件上：添加 keptAlive 标志，patch 时根据此标志调用激活函数； 添加 shouldKeepAlive 标志，unmount 中根据此标志调用失活函数；添加 keepAliveInstance 为对应的 KeepAlive 组件，以便找到失活激活函数
  3. 缓存使用最新一次访问算法
